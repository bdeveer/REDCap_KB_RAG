## Expression Type
Calculated field (field type = `calc`) on the Screening event in a longitudinal project.

## Errors Found

### Syntax Errors

**`datediff()` missing required third argument (unit string)**
The `datediff()` function requires exactly 3 arguments: two dates and a unit string. Your formula provides only two arguments: `datediff([enroll_date], [fu_date])`.

The unit string must be one of: `'d'` (days), `'M'` (months), `'y'` (years), `'h'` (hours), `'m'` (minutes), `'s'` (seconds). Since you're calculating days to follow-up, use `'d'`.

### Reference Errors

**`fu_date` field location unclear**
Your project structure shows that `fu_date` is assigned to the Screening event only—the same event where this calculated field lives. The formula references it as `[fu_date]` with no event prefix, which should resolve correctly at runtime because both are on the same event.

However, the field name `fu_date` (follow-up date) suggests it logically belongs on the followup event, not the screening event. If `fu_date` is actually stored on the `followup_arm_1` event instead, the reference must use the cross-event syntax: `[followup_arm_1][fu_date]`.

**Action:** Verify in your Project Setup → Designate Instruments for My Events that `fu_date` is assigned to which event(s). If it's on followup_arm_1, update the formula accordingly.

## Corrected Expression

If `fu_date` is on the Screening event (same as the calculated field):
```
if([enroll_date] <> '', datediff([enroll_date], [fu_date], 'd'), '')
```

If `fu_date` is on the followup_arm_1 event:
```
if([enroll_date] <> '', datediff([enroll_date], [followup_arm_1][fu_date], 'd'), '')
```

## Additional Notes

The `if()` wrapper returns an empty string when `enroll_date` is blank, which is correct. The `datediff()` result will be a non-negative integer (number of days between enrollment and follow-up). If you need signed values (negative when fu_date is earlier than enroll_date), add a fourth argument: `datediff([enroll_date], [fu_date], 'd', true)`.
